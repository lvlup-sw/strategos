## 1. Problem

Slice A (`Strategos.Contracts` 0.2.0 — events / workflow IR / invariant catalog) shipped via
`#99/#101/#102/#103`. The 2.8.0 milestone has 10 open issues that, taken together, are the
*remaining* close-out surface. They split into three coherent units plus two administrative
dispositions — a deliberate **3-PR close-out**, not ten independent tasks.

The load-bearing gap: the **semantic-merge-queue cross-repo contract** (the payload exarchos's
CI coordinator hands to basileus's AgentHost and back) has no TypeSpec source of truth yet.
Grep confirms `MergeGateDecision` / `JourneyResult` / `WorkflowRef` / `WorkflowCatalog` /
`degraded_reason` are absent from `src/Strategos.Contracts/**/*.tsp`. Separately, the
Ontology MCP library emits descriptors but is not SDK-registrable (#104), and two #51/#52
follow-ups leave drift surfaces un-gated (#105, #106, #107).

- **#104 has a consumer-proven resolution.** Valkyrie (#72–#75) already hand-rolled the

- **#54 is mostly done.** All four slice-B children (#50, #51, #52, #53) are closed. The

## 3. PR 1 — Cross-product contract surface (S1–S4)

The milestone's namesake. TypeSpec is the single source; the build emits JSON Schema (exarchos
derives Zod) + C# records (basileus consumes the DLL). Built in dependency order:

**S2 (#64) — base response envelope.** New `IntentEnvelope` (or `ResponseMeta` block) in
`Strategos.Contracts` carrying `_meta { degraded: boolean, degraded_reason?: DegradedReason }`
and the shared `_perf` block. `DegradedReason` is a closed enum (downstream-unreachable,
judge-timeout, malformed-output, …). Every intent response extends this **one** base — no
per-envelope `_meta` forks (DIM-3 single contract; INV-7 immutable record). The consumer rule
— `degraded == true` short-circuits any "use the decision" logic — is the contract, and
`degraded` *is* the DIM-2 observability seam (fallback is visible, never silent).

**S3 (#66) — `WorkflowRef` discriminated union.** Discriminator `kind`: `"catalog"`
(`workflow_id` + `catalog_version`) and `"authored"` (`journey_description`, deferred
operationally). Language-neutral monikers, not CLR types — correct for a wire contract crossing
into basileus C# and exarchos TS (INV-8 polyglot identity). Open/closed in ref-space (DIM-6):
a new ref kind is one schema addition.

**S4 (#65) — `WorkflowCatalog`.** Manifest + array of `WorkflowIR` entries; the publishing
pipeline lets an exarchos `.workflow.ts` change produce a new catalog version. Routes through
Contracts versioning/generators — image-bake/runtime-fetch alternatives rejected (DIM-1 single
distribution surface). `catalog_version` is what #66's `catalog` variant pins.

**S1 (#63) — `MergeGateDecision` + `JourneyResult`.** Two envelopes extending the S2 base,
sharing the discriminated `next_actions` pattern; `suggested_journeys` / `journey_outcomes` are
typed `WorkflowRef[]` (consumes S3). The load-bearing cross-repo payload.

**Acceptance:** TS + C# bindings generated and published; cross-product round-trip green against
the #53 fixture corpus in **both** languages (DIM-4 — tests construct bindings the way consumers
do); catalog refs validate against the S4 manifest at consumer time.

## 4. PR 3 — Ontology MCP registration bridge (#104)

Resolve the INV-2 ↔ INV-3 tension with the consumer-proven adapter, promoted to first-party.
`OntologyToolDiscovery` stays descriptor-first (INV-2 — library, not server; ontology remains
analyzer-only, **no** source generator added). Add:

- `OntologyToolDiscovery.CreateServerTools(graph)` → `IEnumerable<McpServerTool>`, each built via
  `McpServerTool.Create(...)` from a descriptor, preserving `OutputSchema` + `ToolAnnotations` +
  `ConstraintSummaries`;
- `IMcpServerBuilder.AddOntologyTools(graph)` extension wrapping `WithTools(IEnumerable<McpServerTool>)`.

To keep the **core** assembly SDK-free (INV-2), the SDK-dependent adapter lands in a thin
companion package (`Strategos.Ontology.MCP.Hosting` or equivalent) — the core continues to emit
descriptors with no `ModelContextProtocol` type-level dependency. Rejected: `[McpServerTool]`
attributes on the four tool classes (SDK reflection drops graph-enriched schemas — DIM-3 loss —
and couples the core to the SDK — INV-2 strain); source-gen (violates INV-2 analyzer-only).

**Acceptance (INV-3, DIM-3, DIM-4):** round-trip test asserts each registered `Tool` preserves
`OutputSchema` + `ToolAnnotations`; `tools/list` advertises all four; `tools/call` routes
`ontology_query` etc. Tracks the latest MCP spec.

## 5. PR 2 — Drift / AGWF hardening

Three small follow-ups closing drift surfaces left by #51/#52, plus the #54 close.

- **#105 — AGWF catalog parity test.** In `Strategos.Generators.Tests`, assert for each of the
  10 live codes that catalog `severity`/`summary`/`remediation` matches the
  `WorkflowDiagnostics` descriptor (`defaultSeverity`/`title`/`messageFormat`). Closes the
  silent-divergence gap between hand-authored `AgwfCatalog.tsp` and runtime descriptors
  (INV-5; DIM-4 test fidelity, DIM-3).
- **#107 — drift-issue label fail-open.** `scripts/build-drift-issue.sh`: idempotent
  `gh label create cross-product:strategos … || true` before `gh issue create`, so a fresh
  exarchos environment can file the drift issue (DIM-2 — the notify path stops failing silently).
- **#106 — api-mirror scope.** Confirm whether exarchos's `strategos-api-mirror.test.ts` parses
  only the 7 gated entrypoints or the transitive type graph. If transitive, the 5 ungated
  continuations (`IForkPathBuilder`, `ILoopForkJoinBuilder`, `IApprovalEscalationBuilder`,
  `IApprovalRejectionBuilder`, `IContextBuilder`) are a cross-product blind spot — **expand the
  gated set**; if only the 7, **document the boundary** in `public-api-drift.yml`. Decision is
  evidence-gated on the exarchos test (INV-5; DIM-3).
- **#54 — verify and close.** Confirm slice-B children closed and the exarchos-side gate items
  green, then close administratively. No code.
