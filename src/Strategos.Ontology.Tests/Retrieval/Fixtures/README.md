# Retrieval test fixtures

## `qdrant-dbsf-oracle.json`

Parity oracle for `RankFusion.DistributionBased`. Generated by
`scripts/regenerate-dbsf-oracle.py` against the Qdrant 2024 DBSF reference
implementation. **Manual run; not in CI.** Regenerate after bumping the
`qdrant-client` pin in `scripts/requirements.txt` or after adding/removing
fixture queries:

```bash
python3 -m pip install -r scripts/requirements.txt
python3 scripts/regenerate-dbsf-oracle.py
```

The C# test `RankFusionDistributionBasedTests.DistributionBased_AgainstQdrantOracle_AllQueriesMatch_Within1eMinus9`
loads this JSON and asserts per-document fused scores match the oracle within
`1e-9`. The tolerance (vs. the `1e-12` used for the Reciprocal/Cormack parity)
absorbs the minor float-ordering differences between the Python and C#
implementations of the μ±3σ normalization.
